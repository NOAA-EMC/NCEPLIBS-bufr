4 C> THIS SUBROUTINE IS USED TO SPECIFY A VALUE TO BE WRITTEN
5 C> INTO A SPECIFIED LOCATION WITHIN SECTION 0 OR SECTION 1 OF ALL BUFR
6 C> MESSAGES THAT WILL BE OUTPUT BY FUTURE CALLS TO ANY OF THE BUFR
7 C> ARCHIVE LIBRARY SUBROUTINES WHICH CREATE SUCH MESSAGES (E.G. WRITCP,
8 C> WRITSB, COPYMG, WRITSA, ETC.). IT WILL WORK ON ANY MESSAGE ENCODED
9 C> USING BUFR EDITION 2, 3 OR 4, AND IT CAN BE CALLED AT ANY TIME,
10 C> INCLUDING BEFORE THE FIRST CALL TO BUFR ARCHIVE LIBRARY SUBROUTINE
11 C> OPENBF IF IT IS DESIRED FOR THE NEW VALUE TO ALSO BE INCLUDED IN ANY
12 C> DX DICTIONARY TABLE MESSAGES THAT WILL BE OUTPUT BY BUFR ARCHIVE
13 C> LIBRARY SUBROUTINE WRITDX. IN ANY CASE, THE LOCATION WITHIN WHICH
14 C> TO STORE THE VALUE IS SPECIFIED VIA THE MNEMONIC S01MNEM, AS
15 C> EXPLAINED IN FURTHER DETAIL BELOW. IF MULTIPLE VALUES ARE DESIRED
16 C> TO BE CHANGED WITHIN SECTION 0 OR SECTION 1 OF FUTURE OUTPUT
17 C> MESSAGES, THEN EACH SUCH VALUE (AND CORRESPONDING LOCATION)
18 C> SHOULD BE SPECIFIED USING A SEPARATE CALL TO THIS SUBROUTINE.
19 C> NOTE THAT EACH CALL TO THIS SUBROUTINE WITH A PARTICULAR LOCATION
20 C> SPECIFICATION WILL OVERRIDE THE EFFECT OF ANY PREVIOUS CALL WITH
21 C> THAT SAME SPECIFICATION (OR, IN THE CASE OF THE FIRST CALL WITH A
22 C> PARTICULAR LOCATION SPECIFICATION, IT WILL OVERRIDE THE DEFAULT
23 C> SECTION 0 OR SECTION 1 VALUE FOR THE CORRESPONDING LOCATION!).
