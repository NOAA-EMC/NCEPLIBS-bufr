subroutine dxdump (   LUNIT,
  LDXOT 
)

THIS SUBROUTINE WRITES, TO LOGICAL UNIT LDXOT, AN ASCII COPY OF THE BUFR DICTIONARY TABLE INFORMATION ASSOCIATED WITH THE BUFR FILE DEFINED BY LOGICAL UNIT LUNIT.

IT IS ESPECIALLY USEFUL FOR DETERMINING THE CONTENTS OF ARCHIVE BUFR FILES WHICH MAY HAVE SUCH INFORMATION EMBEDDED AS DX MESSAGES AT THE FRONT OF THE FILE. THE OUTPUT FILE WILL BE IN A FORMAT SUITABLE FOR SUBSEQUENT INPUT AS A USER-DEFINED DICTIONARY TABLES FILE TO BUFR ARCHIVE LIBRARY SUBROUTINE OPENBF, AND IN THAT SENSE THIS SUBROUTINE CAN BE VIEWED AS THE LOGICAL INVERSE OF BUFR ARCHIVE LIBRARY SUBROUTINE RDUSDX. NOTE THAT THE BUFR FILE ASSOCIATED WITH LOGICAL UNIT LUNIT MUST HAVE ALREADY BEEN IDENTIFIED TO THE BUFR ARCHIVE LIBRARY SOFTWARE VIA A PRIOR CALL TO OPENBF.

PROGRAM HISTORY LOG: 2004-08-18 J. ATOR – ORIGINAL AUTHOR 2007-01-19 J. ATOR – CORRECTED OUTPUT FOR REFERENCE VALUES LONGER THAN 8 DIGITS 2014-12-10 J. ATOR – USE MODULES INSTEAD OF COMMON BLOCKS

USAGE: CALL DXDUMP (LUNIT, LDXOT) INPUT ARGUMENT LIST: LUNIT - INTEGER: FORTRAN LOGICAL UNIT NUMBER FOR BUFR FILE LDXOT - INTEGER: FORTRAN LOGICAL UNIT NUMBER FOR OUTPUT FILE

INPUT FILES: UNIT "LUNIT" - BUFR FILE WITH EMBEDDED DX DICTIONARY MESSAGES

OUTPUT FILES: UNIT "LDXOT" - ASCII VERSION OF DX DICTIONARY INFORMATION, IN FORMAT SUITABLE FOR SUBSEQUENT INPUT TO OPENBF

REMARKS: THIS ROUTINE CALLS: BORT NEMTBD STATUS STRSUC THIS ROUTINE IS CALLED BY: None Normally called only by application programs.
