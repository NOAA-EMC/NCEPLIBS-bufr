4 C> THIS SUBROUTINE WRITES OR READS SPECIFIED VALUES TO OR FROM
5 C> THE CURRENT BUFR DATA SUBSET WITHIN INTERNAL ARRAYS, WITH THE
6 C> DIRECTION OF THE DATA TRANSFER DETERMINED BY THE CONTEXT OF
7 C> ABS(LUNIN) (I.E., IF ABS(LUNIN) POINTS TO A BUFR FILE THAT IS OPEN
8 C> FOR INPUT, THEN DATA VALUES ARE READ FROM THE INTERNAL DATA SUBSET;
9 C> OTHERWISE, DATA VALUES ARE WRITTEN TO THE INTERNAL DATA SUBSET).
10 C> THE DATA VALUES CORRESPOND TO MNEMONICS WHICH ARE PART OF A
11 C> DELAYED-REPLICATION SEQUENCE, OR FOR WHICH THERE IS NO REPLICATION
12 C> AT ALL. IF UFBINT IS READING VALUES, THEN EITHER BUFR ARCHIVE
13 C> LIBRARY SUBROUTINE READSB OR READNS MUST HAVE BEEN PREVIOUSLY
14 C> CALLED TO READ THE SUBSET FROM UNIT ABS(LUNIN) INTO
15 C> INTERNAL MEMORY. IF IT IS WRITING VALUES, THEN EITHER BUFR ARCHIVE
16 C> LIBRARY SUBROUTINE OPENMG OR OPENMB MUST HAVE BEEN PREVIOUSLY
17 C> CALLED TO OPEN AND INITIALIZE A BUFR MESSAGE WITHIN MEMORY FOR THIS
18 C> ABS(LUNIN).
19 C>

74 C> SINGLE TABLE D (SEQUENCE) MNEMONIC WITH EITHER 8-

76 C> - IF BUFR FILE OPEN FOR INPUT: THERE ARE THREE

93 C> - IF BUFR FILE OPEN FOR INPUT: NUMBER OF "LEVELS" OF

97 C> OF DATA VALUES WRITTEN TO DATA SUBSET (SHOULD BE
98 C> SAME AS I2)
99 C>
100 C> REMARKS:
101 C> 1) UFBINT CAN ALSO BE CALLED TO PROVIDE INFORMATION ABOUT A SINGLE
102 C> TABLE D (SEQUENCE) MNEMONIC WITH EITHER 8- OR 16-BIT DELAYED
103 C> REPLICATION IN A SUBSET WHEN THE BUFR FILE IS OPEN FOR INPUT.
104 C> THE MNEMONIC IN STR MUST APPEAR AS IT DOES IN THE BUFR TABLE,
105 C> I.E., BRACKETED BY "{" AND "}" OR "[" AND "]" FOR 8-BIT DELAYED
106 C> REPLICATION, OR BRACKETED BY "(" AND ")" FOR 16-BIT DELAYED
107 C> REPLICATION. {NOTE: THIS WILL NOT WORK FOR SEQUENCES WITH
108 C> 1-BIT DELAYED REPLICATION (BRACKETED BY "<" AND ">"), STANDARD
109 C> REPLICATION (BRACKETED BY "'s), OR NO REPLICATION (NO
110 C> BRACKETING SYMBOLS).}
111 C>
112 C> FOR EXAMPLE:
113 C>
114 C> CALL UFBINT(LUNIN,PLEVL,1, 50,IRET,'{PLEVL}')
115 C>
116 C> WILL RETURN WITH IRET EQUAL TO THE NUMBER OF OCCURRENCES OF THE
117 C> 8-BIT DELAYED REPLICATION SEQUENCE PLEVL IN THE SUBSET AND WITH
118 C> (PLEVL(I),I=1,IRET) EQUAL TO THE NUMBER OF REPLICATIONS IN EACH
119 C> OCCURRENCE OF PLEVL IN THE SUBSET. IF THERE ARE NO OCCURRENCES
120 C> OF PLEVL IN THE SUBSET, IRET IS RETURNED AS ZERO.
121 C>
122 C> 2) WHEN THE BUFR FILE IS OPEN FOR OUTPUT, UFBINT CAN BE USED TO
123 C> PRE-ALLOCATE SPACE FOR SOME OR ALL MNEMONICS WITHIN DELAYED
124 C> REPLICATION SEQUENCES. A SUBSEQUENT CALL TO BUFR ARCHIVE
125 C> LIBRARY ROUTINE UFBREP OR UFBSEQ THEN ACTUALLY STORES THE
126 C> VALUES INTO THE BUFR FILES. HERE ARE TWO EXAMPLES OF THIS:
127 C>
128 C> EXAMPLE 1) PROBLEM: AN OUTPUT SUBSET "SEQNCE" IS LAID OUT AS
129 C> FOLLOWS IN A BUFR TABLE (NOTE 16 CHARACTERS HAVE BEEN
130 C> REMOVED FROM THE LAST COLUMN TO ALLOW THE TABLE TO FIT IN
131 C> THIS DOCBLOCK):
132 C>
133 C> | SEQNCE | {PLEVL} |
134 C> | PLEVL | WSPD WDIR TSIG PRLC TSIG PRLC TSIG PRLC |
135 C>
136 C> -- OR --
137 C>
138 C> | SEQNCE | {PLEVL} |
139 C> | PLEVL | WSPD WDIR "PSEQ"3 |
140 C> | PSEQ | TSIG PRLC |
141 C>
142 C> IN THIS CASE THE APPLICATION PROGRAM MUST STORE VALUES WHICH
143 C> HAVE STANDARD REPLICATION NESTED INSIDE OF A DELAYED
144 C> REPLICATION SEQUENCE. FOR EXAMPLE, ASSUME 50 LEVELS OF WIND
145 C> SPEED, WIND DIRECTION, OBSERVED PRESSURE, FIRST GUESS
146 c PRESSURE AND ANALYZED PRESSURE ARE TO BE WRITTEN TO "SEQNCE".
147 C>
148 C> THE FOLLOWING LOGIC WOULD ENCODE VALUES PROPERLY:
149 C>.....................................................................
150 C> ....
151 C> REAL*8 DROBS(2,50)
152 C> REAL*8 SROBS(2,150)
153 C> ....
154 C> DO I=1,50
155 C> DROBS(1,I) = Value of wind speed on level "I"
156 C> DROBS(2,I) = Value of wind direction on level "I"
157 C> SROBS(1,I*3-2) = Value of observed pressure on level "I"
158 C> SROBS(2,I*3-2) = 25. ! Value in Code Table 0-08-021 (TSIG)
159 C> ! for time sigificance (Nominal
160 C> ! reporting time) for observed
161 C> ! pressure on level "I"
162 C> SROBS(1,I*3-1) = Value of first guess pressure on level "I"
163 C> SROBS(2,I*3-1) = 27. ! Value in Code Table 0-08-021 (TSIG)
164 C> ! for time sigificance (First guess)
165 C> ! for first guess pressure on level "I"
166 C> SROBS(1,I*3) = Value of analyzed pressure on level "I"
167 C> SROBS(2,I*3) = 16. ! Value in Code Table 0-08-021 (TSIG)
168 C> ! for time sigificance (Analysis) for
169 C> ! analyzed pressure on level "I"
170 C> ENDDO
171 C>
172 C> ! The call to UFBINT here will not only store the 50
173 C> ! values of WSPD and WDIR into the BUFR subset, it
174 C> ! will also allocate the space to store three
175 C> ! replications of TSIG and PRLC on each of the 50
176 C> ! delayed-replication "levels"
177 C> CALL UFBINT(LUNIN,DROBS,2, 50,IRET,'WSPD WDIR')
178 C>
179 C> ! The call to UFBREP here will actually store the 150
180 C> ! values of both TSIG and PRLC (three values for each
181 C> ! on 50 delayed-replication "levels")
182 C> CALL UFBREP(LUNIN,SROBS,2,150,IRET,'TSIG PRLC')
183 C> ....
184 C> STOP
185 C> END
186 C>.....................................................................
187 C>
188 C> A SIMILAR EXAMPLE COULD BE PROVIDED FOR READING VALUES WHICH
189 C> HAVE STANDARD REPLICATION NESTED WITHIN DELAYED REPLICATION,
190 C> FROM BUFR FILES OPEN FOR INPUT. (NOT SHOWN HERE.)
191 C>
192 C>
193 C> EXAMPLE 2) PROBLEM: AN INPUT SUBSET, "REPT_IN", AND AN OUTPUT
194 C> SUBSET "REPT_OUT", ARE LAID OUT AS FOLLOWS IN A BUFR TABLE
195 C> (NOTE 16 CHARACTERS HAVE BEEN REMOVED FROM THE LAST COLUMN
196 C> TO ALLOW THE TABLE TO FIT IN THIS DOCBLOCK):
197 C>
198 C> | REPT_IN | YEAR MNTH DAYS HOUR MINU {PLEVL} CLAT CLON |

200 C> | PLEVL | PRLC TMBD REHU WDIR WSPD
201 C>
202 C> IN THIS CASE THE APPLICATION PROGRAM IS READING IN VALUES
203 C> FROM A BUFR FILE CONTAINING SUBSET "REPT_IN", CONVERTING
204 C> MONTH AND DAY TO DAY OF YEAR, AND THEN WRITING VALUES TO

206 C> DO THIS IS TO CALL UFBSEQ TO READ IN AND WRITE OUT THE
207 C> VALUES, HOWEVER THIS IS COMPLICATED BY THE PRESENCE OF THE

216 C>.....................................................................
217 C> ....
218 C> REAL*8 OBSI(2000),OBSO(1999),PLEVL(5,255),REPS_8
219 C> CHARACTER*8 SUBSET
220 C> ....
221 C>
222 C> CALL DATELEN(10)
223 C>
224 C> ! Open input BUFR file in LUBFI and open output BUFR file in
225 C> ! LUBFJ, both use the BUFR table in LINDX
226 C> CALL OPENBF(LUBFI,'IN', LINDX)
227 C> CALL OPENBF(LUBFJ,'OUT',LINDX)
228 C>
229 C> ! Read through the BUFR messages in the input file
230 C> DO WHILE(IREADMG(LUBFI,SUBSET,IDATE).GE.0)
231 C>
232 C> ! Open message (for writing) in output file
233 C> CALL OPENMB(LUBFJ,'REPT_OUT',IDATE)
234 C>
235 C> ! Read through the subsets in this input BUFR messages
236 C> DO WHILE(IREADSB(LUBFI).EQ.0)
237 C>

239 C> ! of subset "REPT_IN", storing them into array OBSI

243 C>

245 C> ! replications ("levels") in "PLEVL" for subset
246 C> ! "REPT_IN"" ! {see 1) above in REMARKS}
247 C> CALL UFBINT(LUBFI,REPS_8,1,1,IRET,'{PLEVL}')

251 C> IMO = OBSI(2)

253 C> CALL xxxx(IYR, IMO, IDA, JDY) ! convert month and day
254 C> ! to day of year (JDY)
255 C> OBSO(1) = OBSI(1)
256 C> OBSO(2) = JDY
257 C> DO I = 3,1999
258 C> OBSO(I) = OBSI(1+1)
259 C> ENDDO
260 C>
261 C> PLEVL = GETBMISS()
262 C>
263 C> ! The call to UFBINT here will allocate the space to
264 C> ! later allow UFBSEQ to store IREPS replications of
265 C> ! "PLEVL" into the output BUFR subset "REPT_OUT" (note
266 C> ! here it is simply storing missing values)

268 C> $ 'PRLC TMBD REHU WDIR WSPD')

279 C> ! All done
